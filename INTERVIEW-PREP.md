## SQL — Must Know

| Topic | What to know |
|-------|-------------|
| `SELECT` | Pick which columns to return |
| `WHERE` | Filter rows by condition |
| `INNER JOIN` | Combine two tables on a matching column |
| `LEFT JOIN` | Same as INNER but includes rows with no match (returns NULL) |
| `ORDER BY` | Sort results — ASC (default), DESC (reverse) |
| `GROUP BY` | Group rows for aggregates |
| `COUNT / SUM / AVG` | Aggregate functions used with GROUP BY |
| `INSERT INTO` | Add a new row |
| `UPDATE ... SET ... WHERE` | Change existing rows |
| `DELETE FROM ... WHERE` | Remove rows — always use WHERE or you delete everything |
| `IS NULL / IS NOT NULL` | Check for empty values |
| `ISNULL(value, fallback)` | Return fallback if value is NULL |

### Quick Examples

```sql
-- SELECT with WHERE
SELECT Id, FullName, Status FROM Employees WHERE Status = 'Active';

-- INNER JOIN
SELECT e.FullName, d.Name AS Department
FROM Employees e
INNER JOIN Departments d ON e.DepartmentId = d.Id;

-- LEFT JOIN (includes employees with no department)
SELECT e.FullName, d.Name AS Department
FROM Employees e
LEFT JOIN Departments d ON e.DepartmentId = d.Id;

-- GROUP BY + COUNT
SELECT d.Name, COUNT(e.Id) AS EmployeeCount
FROM Departments d
LEFT JOIN Employees e ON e.DepartmentId = d.Id
GROUP BY d.Name;

-- INSERT
INSERT INTO Departments (Name, Description) VALUES ('Finance', 'Finance team');

-- UPDATE
UPDATE Employees SET Status = 'Terminated' WHERE Id = 5;

-- DELETE
DELETE FROM Employees WHERE Id = 5;
```

## Stored Procedures — Must Know

| Concept | What it does |
|---------|-------------|
| `CREATE PROCEDURE` | Saves SQL as a reusable named block |
| `@Param TYPE = NULL` | Input parameter — default NULL makes it optional |
| `OUTPUT` parameter | Returns a value back to the caller |
| `SET NOCOUNT ON` | Write at the top of every SP — suppresses row count noise |
| `SCOPE_IDENTITY()` | Gets the ID generated by the last INSERT |
| `@@ROWCOUNT` | How many rows were affected by the last statement |
| `THROW 50001, 'msg', 1` | Raise a custom error |
| `BEGIN TRY / CATCH` | Handle errors gracefully |
| `BEGIN TRANSACTION / COMMIT / ROLLBACK` | All-or-nothing operations |
| `DROP PROCEDURE IF EXISTS` | Safe way to recreate a SP |
| `EXEC ProcName @Param` | How to call a stored procedure |

### SP Templates

```sql
-- Basic SP with optional filter
CREATE PROCEDURE GetEmployeesByDept
    @DepartmentId INT = NULL
AS
BEGIN
    SET NOCOUNT ON;

    SELECT FullName, Designation, Status
    FROM Employees
    WHERE (@DepartmentId IS NULL OR DepartmentId = @DepartmentId)
    ORDER BY FullName;
END;


-- SP with INSERT + OUTPUT param (returns new ID)
CREATE PROCEDURE CreateDepartment
    @Name        NVARCHAR(100),
    @NewId       INT OUTPUT
AS
BEGIN
    SET NOCOUNT ON;

    INSERT INTO Departments (Name) VALUES (@Name);
    SET @NewId = SCOPE_IDENTITY();
END;


-- SP with transaction + error handling
CREATE PROCEDURE TransferEmployee
    @EmployeeId   INT,
    @NewDeptId    INT
AS
BEGIN
    SET NOCOUNT ON;
    BEGIN TRY
        BEGIN TRANSACTION;

            IF NOT EXISTS (SELECT 1 FROM Employees WHERE Id = @EmployeeId)
                THROW 50001, 'Employee not found.', 1;

            UPDATE Employees SET DepartmentId = @NewDeptId WHERE Id = @EmployeeId;

        COMMIT TRANSACTION;
    END TRY
    BEGIN CATCH
        ROLLBACK TRANSACTION;
        THROW;
    END CATCH
END;


-- How to call SPs
EXEC GetEmployeesByDept @DepartmentId = 3;
EXEC GetEmployeesByDept;                    -- no filter, returns all

DECLARE @NewId INT;
EXEC CreateDepartment @Name = 'Finance', @NewId = @NewId OUTPUT;
SELECT @NewId;                              -- prints the new ID
```

## C# — Must Know

| Concept | What to know |
|---------|-------------|
| Class + properties | `public string Name { get; set; }` |
| Constructor | Inject dependencies through constructor |
| Interface | Contract — defines what, not how |
| `async` / `await` | Non-blocking — thread doesn't wait idle |
| `List<T>` | Generic list — create, loop, add |
| `foreach` | Loop through a collection |
| String methods | Trim, ToUpper, Contains, Replace, Split |
| `try / catch` | Handle exceptions |
| LINQ | Where, Select, FirstOrDefault, Count |
| Null handling | `??`, `?.`, null checks |

### C# Templates

```csharp
// Class with properties
public class Employee
{
    public int Id { get; set; }
    public string FullName { get; set; } = string.Empty;
    public string Status { get; set; } = "Active";
    public decimal BasicSalary { get; set; }
}


// Interface
public interface IEmployeeService
{
    Task<List<Employee>> GetAllAsync();
    Task<Employee?> GetByIdAsync(int id);
}


// Class implementing interface + constructor injection
public class EmployeeService : IEmployeeService
{
    private readonly AppDbContext _context;

    public EmployeeService(AppDbContext context)
    {
        _context = context;
    }

    public async Task<List<Employee>> GetAllAsync()
    {
        return await _context.Employees.ToListAsync();
    }

    public async Task<Employee?> GetByIdAsync(int id)
    {
        return await _context.Employees.FirstOrDefaultAsync(e => e.Id == id);
    }
}


// LINQ examples
var active = employees.Where(e => e.Status == "Active").ToList();
var names  = employees.Select(e => e.FullName).ToList();
var first  = employees.FirstOrDefault(e => e.Id == 5);
var count  = employees.Count(e => e.Status == "Active");


// String methods
string name = "  hello world  ";
name.Trim();                     // "hello world"
name.ToUpper();                  // "HELLO WORLD"
name.Contains("hello");          // true
name.Replace("hello", "hi");     // "hi world"
name.Split(' ');                 // ["hello", "world"]


// try / catch
try
{
    int result = int.Parse("abc"); // throws FormatException
}
catch (FormatException ex)
{
    Console.WriteLine($"Error: {ex.Message}");
}


// Null handling
string? value = null;
string result = value ?? "default";   // "default"
int? length = value?.Length;          // null (no crash)
```

## OOP Concepts — If Asked

| Term | One-line answer |
|------|----------------|
| **Class** | Blueprint for creating objects |
| **Interface** | Contract — says what methods must exist, not how |
| **Inheritance** | One class takes all properties/methods of another (`class B : A`) |
| **Encapsulation** | Hide internal details, expose only what's needed (`private` / `public`) |
| **Polymorphism** | Same method name, different behaviour depending on the object |

### Answer to #10 (tricky one)

```sql
-- WHERE filters rows BEFORE grouping
SELECT DepartmentId, COUNT(*) AS Total
FROM Employees
WHERE Status = 'Active'        -- filters individual rows first
GROUP BY DepartmentId;

-- HAVING filters AFTER grouping
SELECT DepartmentId, COUNT(*) AS Total
FROM Employees
GROUP BY DepartmentId
HAVING COUNT(*) > 5;           -- filters groups (departments with more than 5)
```

## Key Vocabulary — Quick Answers

| They ask | You say |
|----------|---------|
| What is a stored procedure? | Pre-compiled SQL saved in the database, called by name with parameters |
| What is a transaction? | A group of statements that all succeed or all roll back together |
| What is SCOPE_IDENTITY()? | Returns the ID generated by the last INSERT in the current scope |
| What is SET NOCOUNT ON? | Stops SQL Server sending row count messages — improves SP performance |
| What is an interface in C#? | A contract — defines what methods a class must have, not how |
| What is async/await? | Non-blocking execution — thread does other work while waiting for DB |
| What is dependency injection? | Passing dependencies through the constructor instead of creating them inside |
| INNER vs LEFT JOIN? | INNER: only matching rows. LEFT: all rows from left table, NULL if no match |
| WHERE vs HAVING? | WHERE filters rows before grouping. HAVING filters after grouping |
